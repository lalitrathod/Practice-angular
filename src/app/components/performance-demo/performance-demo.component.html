üîç

OnPush Change Detection

Only run change detection when inputs change or events fire. Reduces overhead significantly for large applications.

  • Use for presentational components
  • Combine with immutable data
  • Default strategy checks entire tree
üìã

TrackBy in *ngFor

Tell Angular how to identify items in lists. Prevents unnecessary DOM recreation when lists change.

  • Use unique property (ID)
  • Avoid using index
  • Huge perf boost for large lists
üì¶

Lazy Loading

Load feature modules only when needed. Reduces initial bundle size and improves load time.

  • Lazy load routes
  • Load components dynamically
  • Code splitting strategies
üîó

Signal-based State

Use Signals instead of Observables for simpler, more performant state management.

  • Fine-grained reactivity
  • Automatic caching
  • No subscription overhead
üß†

Memory Management

Prevent memory leaks by properly cleaning up subscriptions and event listeners.

  • Unsubscribe in ngOnDestroy
  • Use takeUntil operator
  • Clear intervals/timeouts
üìä

Bundle Analysis

Monitor and optimize bundle size. Remove dead code and identify large dependencies.

  • Use webpack-bundle-analyzer
  • Tree-shake unused code
  • Monitor dependencies
üñºÔ∏è

Image Optimization

Optimize images for web. Use responsive images, lazy loading, and modern formats.

  • Use NgSrc directive
  • Lazy load images
  • WebP format support
üìà

Performance Monitoring

Track performance metrics and identify bottlenecks in your application.

  • Use Chrome DevTools
  • Monitor Core Web Vitals
  • Profile change detection

üìä Performance Metrics

Core Web Vitals

  • LCP: Largest Contentful Paint (< 2.5s)
  • FID: First Input Delay (< 100ms)
  • CLS: Cumulative Layout Shift (< 0.1)

Bundle Metrics

  • Bundle Size: Keep main bundle < 150KB
  • Lazy Bundle: Each lazy bundle < 100KB
  • Gzip Size: Measure gzipped size

Runtime Metrics

  • TTI: Time to Interactive
  • FCP: First Contentful Paint
  • FMP: First Meaningful Paint

üéØ Best Practices

  • Use OnPush by default: Switch to Default only if necessary
  • Always use trackBy: Essential for lists with dynamic content
  • Lazy load features: Reduce initial bundle size
  • Prefer Signals: Better performance than RxJS for state
  • Monitor bundle size: Track size in CI/CD pipeline
  • Optimize images: Use responsive images and lazy loading
  • Clean up subscriptions: Prevent memory leaks
  • Profile regularly: Use Chrome DevTools Performance tab
  • Test on slow devices: Ensure good performance everywhere
  • Use service workers: Cache assets for offline support
