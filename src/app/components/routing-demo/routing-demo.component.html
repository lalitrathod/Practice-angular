What is Angular Routing?

Angular Router enables navigation from one view to the next as users perform application tasks. It maps URL paths to components and allows for deep linking, lazy loading, and route guards.

Q: What is the difference between routerLink and Router.navigate()?

A: routerLink is a directive used in templates for declarative navigation. Router.navigate() is used programmatically in component code. Both achieve the same result, but routerLink is preferred for template-based navigation.

Q: What are Route Guards?

A: Route guards are interfaces that control navigation. Types include:

  • CanActivate: Controls if route can be activated
  • CanDeactivate: Controls if route can be deactivated
  • CanLoad: Controls if module can be loaded (lazy loading)
  • Resolve: Resolves data before route activation

Q: What is Lazy Loading?

A: Lazy loading loads feature modules only when their routes are accessed, reducing initial bundle size and improving app startup time. Use loadChildren instead of component in route configuration.

Q: How do you pass data between routes?

A: Multiple ways:

  • Route Parameters: /user/:id
  • Query Parameters: /products?page=1
  • Route State: router.navigate(['/path'], {{ '{' }} state: {{ '{' }} data {{ '}' }} {{ '}' }})
  • Service: Shared service for complex data

Q: What is the difference between ActivatedRoute and Router?

A:

  • ActivatedRoute: Contains info about the current route (params, queryParams, data)
  • Router: Service for navigation and route configuration
